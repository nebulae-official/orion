# Design Patterns

- **Repository:** Abstract data access behind interfaces.
- **Factory:** Create providers/clients via factories, not raw constructors.
- **Strategy:** Swap implementations (e.g. LOCAL/CLOUD) via strategy interfaces.
- **Dependency Injection:** Inject dependencies; do not import singletons.
- **Observer:** Use events/pub-sub for cross-service communication (Redis).
